# Development Workflows

This directory contains workflow definitions for common development tasks.

## Purpose

Workflows define multi-step processes for:

- Feature development
- Bug fixing
- Code review
- Release preparation

## Format

Each workflow should be a `.md` file containing:

- Workflow name and purpose
- Step-by-step instructions
- Decision points
- Expected outcomes

## Common Workflows

### Feature Development

1. Create feature branch
2. Implement changes
3. Write tests
4. Update documentation
5. Submit for review

### Bug Fix

1. Reproduce issue
2. Identify root cause
3. Implement fix
4. Add regression test
5. Update changelog
